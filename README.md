# מי ומה 🟧

לוח פרסום בפיקסלים לפי קטגוריות. כל אחד מעלה תמונה + קישור, את מאשרת ושולחת קישור תשלום בוואטסאפ, ואחרי התשלום המודעה עולה לאתר.

האתר בנוי ב-**React + Vite** ומחובר ל-**Supabase** (מסד נתונים + אחסון תמונות + התחברות מנהל). הכל בחבילה החינמית.

---

## מה צריך פעם אחת (15–20 דקות)

### שלב 1 — חשבון Supabase
1. היכנסי ל-https://supabase.com והירשמי (חינם).
2. לחצי **New project**. תני שם (למשל `mi-uma`), בחרי סיסמה למסד הנתונים, ואזור Frankfurt/Europe.
3. חכי דקה–שתיים עד שהפרויקט מוכן.

### שלב 2 — הרצת ה-SQL
1. בתפריט הצד: **SQL Editor → New query**.
2. פתחי את הקובץ `supabase-setup.sql` שבפרויקט, העתיקי את **כל** התוכן, הדביקי, ולחצי **Run**.
3. אמור להופיע "Success". זה יצר את הטבלה, התצוגה, ודלי התמונות.

### שלב 3 — הגדרת המנהלת (זה את)
1. בקובץ `supabase-setup.sql`, בשורה האחרונה, החליפי את `YOUR-ADMIN-EMAIL@example.com` באימייל שאיתו **תירשמי לאתר** (כפתור "התחברות" → "פתיחת חשבון"). אם כבר הרצת את ה-SQL — פשוט הריצי שוב רק את השורה הזו עם האימייל הנכון.
2. אחרי שהאתר באוויר: היכנסי אליו, לחצי **התחברות → פתיחת חשבון**, והירשמי עם **בדיוק אותו אימייל**. מאותו רגע יופיע לך כפתור **"ניהול"**.

> מפרסמים רגילים נרשמים לבד דרך האתר. רק האימייל שברשימת ה-admins מקבל גישת ניהול.

### שלב 3.5 — כתובות אימות (חשוב לשחזור סיסמה)
ב-Supabase: **Authentication → URL Configuration**. תחת **Site URL** הזיני את כתובת האתר ב-Vercel (אחרי שתקבלי אותה), ותחת **Redirect URLs** הוסיפי אותה גם. בלי זה קישור איפוס הסיסמה לא יחזיר את המשתמש לאתר.

### שלב 4 — העתקת המפתחות
1. בתפריט: **Project Settings → API**.
2. העתיקי לצד שני ערכים:
   - **Project URL** (משהו כמו `https://abcd.supabase.co`)
   - **anon public** key (מפתח ארוך)
   - *(המפתח הזה נועד להיות פומבי — הוא מוגן ע"י ההרשאות שהגדרנו, אז זה בטוח.)*

## העלאה ל-GitHub + Vercel

### שלב 5 — GitHub
1. צרי חשבון ב-https://github.com אם אין לך.
2. צרי **New repository** (למשל `mi-uma`), פרטי או ציבורי — לא משנה.
3. העלי את **כל הקבצים מהתיקייה הזו** (אפשר בגרירה לדף ה-repo: "uploading an existing file" → לגרור את כל הקבצים).
   - ⚠️ אל תעלי את התיקייה `node_modules` ואת קובץ `.env` (ה-`.gitignore` כבר מטפל בזה).

### שלב 6 — Vercel
1. היכנסי ל-https://vercel.com והירשמי עם חשבון ה-GitHub.
2. **Add New → Project**, בחרי את ה-repo `mi-uma`, ולחצי **Import**.
3. Vercel יזהה לבד שזה Vite — לא צריך לשנות הגדרות build.
4. לפני Deploy פתחי **Environment Variables** והוסיפי שני משתנים:

   | Name | Value |
   |---|---|
   | `VITE_SUPABASE_URL` | ה-Project URL מ-Supabase |
   | `VITE_SUPABASE_ANON_KEY` | ה-anon key מ-Supabase |

5. לחצי **Deploy**. אחרי דקה האתר באוויר 🎉 (תקבלי כתובת `something.vercel.app`).

> אם שינית משתני סביבה אחרי הפריסה — צריך **Redeploy** כדי שייכנסו לתוקף.

## איך זה עובד ביום-יום

**מבקר/מפרסם:** בוחר קטגוריה → רואה לוח עם **משבצות פנויות בצבעי פסטל** (כל אחת עם כמות פיקסלים ומחיר) → לוחץ על משבצת פנויה → **נרשם/מתחבר** → מעלה תמונה, קישור וטלפון → שולח. בכל שלב הוא יכול להיכנס ל**"האזור שלי"** כדי לעקוב, לערוך או להסיר מודעה, וגם לאפס סיסמה.

**את (אזור ניהול):** מתחברת עם חשבון המנהלת ולוחצת **"ניהול"**. יש 5 לשוניות:
- **לבדיקה** — מודעות חדשות → *אישור תוכן* / *דחייה*.
- **ממתין לתשלום** — **"שלח לתשלום"** (וואטסאפ עם הודעה מוכנה; מדביקה קישור Grow). כששילם → **"שולם · העלה"** (כאן נקבע מועד הפרסום והתוקף לשנה).
- **עדכונים** — שינויים שמפרסמים ביקשו: *נוכחי מול מבוקש* → *אשר* / *דחה*. המודעה הישנה נשארת עד אישור.
- **באוויר** — מודעות חיות (רואים מועד פרסום וחלון החזר).
- **הוסרו** — מודעות שמפרסמים הסירו מרצונם (לרשומות; אפשר למחוק לצמיתות).

**כללי תוקף והחזר** (מופיעים גם ב"תנאי שימוש"):
- **תוקף מודעה: שנה לפחות** ממועד הפרסום, אלא אם המפרסם בוחר להסיר ולוותר על המקום.
- **ביטול בהחזר: עד 14 יום** מההזמנה. אחרי כן אין החזר. (את מבצעת את ההחזר בפועל דרך Grow; המערכת מראה לך אם המודעה עדיין בחלון ה-14 יום.)

> 💡 **תשלום:** ב-Grow פותחים *בקשת תשלום* על הסכום, מעתיקים את הלינק, ומדביקים בהודעת הוואטסאפ. אין צורך באינטגרציה.

> ✉️ **מיילים (אישור הרשמה ושחזור סיסמה):** עובדים מהקופסה לנפחים קטנים. אם בא לך שמפרסמים יוכלו לפרסם מיד בלי לאשר מייל — כבי את **Confirm email** ב-Supabase (Authentication → Providers → Email). לשחזור סיסמה צריך שהמייל יעבוד. להיקף גדול הגדירי SMTP משלך (Authentication → Emails).

> 💡 **תשלום:** ב-Grow פותחים *בקשת תשלום / דף תשלום* על הסכום הנכון, מעתיקים את הלינק, ומדביקים בהודעת הוואטסאפ. אין צורך בשום אינטגרציה.

## דברים טובים לדעת

- **מחיר:** ₪1 לפיקסל. חבילת 10,000 פיקסלים = ₪10,000. אם נראה לך גבוה — אפשר לשנות את הקבוע `PRICE` בקובץ `src/App.jsx`.
- **קטגוריות:** מוגדרות במערך `CATEGORIES` ב-`src/App.jsx` — קל להוסיף/להסיר.
- **בטיחות קישורים:** יש בדיקה בסיסית שמסמנת קישורים חשודים בדגל ⚠️ לתשומת לבך. ההגנה האמיתית היא שאת מאשרת כל מודעה ידנית.
- **תמונות:** נדחסות אוטומטית לגודל הבלוק לפני העלאה — אז גם תמונה כבדה נשמרת קטנטנה.
- **פרטי קשר:** ערכי את האימייל והטלפון בעמוד "צור קשר" — במשתנה `CONTACT` בראש הקובץ `src/App.jsx`.
- **תנאי שימוש:** עמוד "תנאי שימוש" הוא **תבנית בסיסית** — החליפי את הפרטים בסוגריים המרובעים, ומומלץ שעו"ד יעבור עליו לפני שמתחילים לגבות תשלום.
- **מדיניות פרטיות:** עמוד "מדיניות פרטיות" הוא גם תבנית — מלאי את פרטי העסק והספקים (Supabase/Vercel/Grow) ובדקי מול עו"ד.
